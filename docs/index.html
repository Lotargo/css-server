Static runtime experiment

CSS as a compute layer, not just paint.

CSS-Server tests how far modern browser CSS can be pushed: DOM nodes hold task state, typed custom properties carry values, selectors route states, and CSS resolves parts of the computation in the open.

Proof module

A calculator is small enough to inspect and large enough to fail honestly.

Arithmetic, modes, converters, graphing, settings, history, and diagnostics create enough state to prove whether CSS can carry real deterministic behavior. This is why the demo is a calculator: it exposes the runtime instead of hiding it behind a toy counter.

Standalone page

What this is

A runtime boundary test, not a claim that GitHub Pages runs servers.

There are almost no projects like this because CSS is designed as a styling language. It has no sockets, filesystem, durable processes, request listeners, private environment variables, or normal control flow. CSS-Server explores the narrow but interesting space that remains: local computation inside the browser, with JavaScript and Rust reduced to I/O bridges where CSS cannot act.

GitHub Pages

GitHub Pages serves files only

The public landing has no inbound backend, no hidden Node process, and no server-side execution on GitHub infrastructure.

Browser runtime

CSS and HTML carry visible state

The browser resolves typed attributes, custom properties, calc(), and selector-driven classification locally.

Local app

Tauri/Rust is the desktop bridge

The downloadable/local version can expose an HTTP endpoint, SQLite history, and IPC. That is outside the GitHub Pages boundary.

Static fetch boundary

API checks here are outbound browser requests.

This form does not prove that Pages has a backend. It proves the opposite boundary: a static page can make user-triggered fetch() calls, then reflect status, latency, payload size, body presence, and content type back into DOM attributes for CSS classification.

Do not paste shared API secrets here. Browser requests are visible to the user and constrained by CORS.

What CSS handles

  • Numeric values through custom properties and typed attributes.
  • Branching and status classification through selectors and style conditions.
  • Visible task cells, so runtime state can be inspected in the DOM.

What CSS cannot be

  • An inbound HTTP server on GitHub Pages.
  • A place for private API keys, secrets, or provider credentials.
  • A replacement for JavaScript/Rust when durable I/O is required.

Project shape

One project, two boundaries.

The GitHub Pages landing is the public static shell. The local Tauri build is the heavier proof: Rust accepts HTTP requests, SQLite stores history, JavaScript observes browser events, and CSS remains the computation substrate where the experiment allows it.
