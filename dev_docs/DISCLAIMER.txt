ОФИЦИАЛЬНЫЙ ДИСКЛЕЙМЕР И АРХИТЕКТУРНОЕ УВЕДОМЛЕНИЕ

Проект: CSS-Server (Glass Box Backend Architecture) Статус: Экспериментальный
концепт / PoC (Proof of Concept)

Настоящий документ составлен для фиксации технических, идеологических и
практических ограничений архитектуры «CSS-Server». Любому разработчику,
инженеру или инвестору, изучающему данный концепт, настоятельно рекомендуется
ознакомиться с нижеизложенными пунктами во избежание фатальных архитектурных
ошибок.

1. Суть эксперимента (The Vision)

Проект CSS-Server исследует возможность использования браузерного движка
(Blink/WebKit) в качестве среды для исполнения серверной бизнес-логики. Мы
утверждаем, что с внедрением новейших стандартов CSS (функция if(),
продвинутый attr(), диапазонные запросы style(), псевдокласс :has()),
связка HTML+CSS стала Тьюринг-полной декларативной вычислительной средой,
способной выполнять математические операции, ветвление (if/else) и
маршрутизацию без участия JavaScript в бизнес-логике. В этой
парадигме DOM-дерево выступает оперативной памятью (ОЗУ) и AST-деревом,
а CSS — процессором (ALU).

2. Уведомление о непригодности для Production (The Reality Check)

Мы официально заявляем, что текущая реализация КАТЕГОРИЧЕСКИ НЕ ПРЕДНАЗНАЧЕНА
для использования в реальных коммерческих продуктах (Production),
высоконагруженных системах или сервисах, требующих надежности и
безопасности. Проект находится в зоне «Тьюринговской трясины» (Turing
Tarpit) и имеет следующие фундаментальные ограничения:

  - Катастрофическая производительность: Вычисления производятся через
    графический конвейер браузера (Reflow/Repaint). Это на порядки
    медленнее традиционных языков (Go, Rust, Node.js).
  - Угроза переполнения памяти (OOM): Каждый запрос конвертируется в тяжелый
    DOM-узел. При малейшем скачке трафика процесс исчерпает лимиты ОЗУ.
  - Отсутствие алгоритмической базы: В CSS отсутствуют встроенные циклы
    (while/for), инструменты манипуляции со строками, побитовые операции и
    криптография.
  - Однопоточность: Движок вычисления стилей блокирует Main Thread, что делает
    параллельную обработку запросов невозможной.

3. Архитектурное табу: Роль системной шины

В проекте присутствует системная прослойка (на базе Tauri/Rust или Node.js/JS),
однако её роль строго ограничена функциями ввода-вывода (I/O). Строго
запрещается переносить логические вычисления, валидацию данных или
маршрутизацию в слой системной шины. Шина является «глухой» — она имеет
право лишь транслировать сетевые запросы в инъекции HTML-кода и считывать
результаты из DOM по завершении CSS-анимаций (тактовых циклов).

4. Стратегическое позиционирование (Троянский конь)

Понимая техническую абсурдность использования браузера в роли сервера, мы
классифицируем данный проект как эзотерическое программирование,
инженерный арт-объект и базис для платформы визуального обучения
(EdTech). Проект создается для того, чтобы:

1.  Визуализировать невидимые серверные процессы (концепция «Стеклянного
    сервера» / Glass Box).
2.  Уничтожить барьер между пониманием фронтенда и бекенда.
3.  Исследовать абсолютные лимиты браузерных движков.

В будущем концепт может быть обернут в высокоуровневую абстракцию
(синтаксический сахар на JS/Python), где код пользователя будет
компилироваться в наш CSS-ассемблер для визуального обучения. Но на текущем
этапе мы сознательно отказываемся от этой абстракции.

5. Текущая фаза разработки (Raw Assembler Phase)

В данный момент проект находится на стадии разработки "голого ядра" (Raw CSS
Logic Core). Мы намеренно не создаем удобных инструментов для разработчика. Наша
текущая цель — спроектировать базовые логические вентили (AND, OR, NOT),
сумматоры, тактовые генераторы и маршрутизаторы исключительно силами
каскадных таблиц стилей и HTML-разметки. Мы изучаем физику браузера, чтобы
доказать жизнеспособность метода на самом низком уровне.
