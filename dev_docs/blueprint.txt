# Блюпринт: CSS-Server v1.0
**Концептуальная архитектура Реактивной Графовой Вычислительной Машины (Glass Box Backend)**

## 1. Фундаментальная философия
Архитектура **CSS-Server** опирается на классическую Гарвардскую архитектуру вычислительных систем, но использует веб-стандарты в качестве аппаратной абстракции:
*   **HTML (DOM-дерево) — это Оперативная память и AST.** Состояние системы хранится в узлах, кастомных атрибутах `data-*` и скрытых битовых регистрах (`<input type="checkbox">`).
*   **CSS — это Процессор (ALU) и Маршрутизатор.** Новейшие спецификации CSS 2025/2026 годов позволяют проводить паттерн-матчинг, извлекать типизированные данные из DOM и применять условную математическую и булеву логику.
*   **Tauri / Rust (или Node.js) — это Контроллер Ввода-Вывода (Системная шина).** У шины нет права на вычисления `if/else`. Её задача — транслировать HTTP/TCP запросы в инъекции HTML-узлов и слушать CSS-анимации для отправки ответов обратно в сеть.

## 2. Архитектура вычислительного пайплайна (с кодом)

Для реализации бекенда мы будем использовать исключительно спецификации **CSS Values and Units Module Level 5** и **CSS Conditional Rules Level 5**.

### Шаг 1: Системная шина инжектирует запрос в Память (HTML)
Когда сервер получает HTTP-запрос `POST /auth {"role": "admin", "age": 22}`, прослойка Tauri создает узел в DOM-дереве. Это наша «выделенная память» под процесс.

```html
<!-- HTML5 (Standard) -->
<div class="http-request" data-endpoint="/auth" data-role="admin" data-age="22">
  <div class="css-processor-core"></div>
</div>
```

### Шаг 2: Извлечение типизированных данных (Advanced attr)
Ранее функция `attr()` в CSS могла возвращать только строки и работала исключительно для псевдоэлементов `::before/::after`. В новых версиях CSS (2025-2026) она научилась парсить типы данных на лету. Мы загружаем данные из памяти в «регистры» CSS-переменных.

```css
/* ПОДДЕРЖКА ВЕРСИЙ: Advanced attr() type() syntax */
/* Chromium 119+ / Baseline 2026 */
.http-request {
  --req-endpoint: attr(data-endpoint type(<string>));
  --req-role: attr(data-role type(<custom-ident>));
  --req-age: attr(data-age type(<number>)); /* Конвертация строки '22' в число 22 для математики */
}
```

### Шаг 3: Маршрутизация (Роутинг) запроса
Традиционный бекенд использует роутеры (Express, FastAPI). В CSS-Server роль маршрутизатора выполняет псевдокласс `:has()`, который сканирует AST-дерево.

```css
/* ПОДДЕРЖКА ВЕРСИЙ: псевдокласс :has() */
/* Chromium 105+, Safari 15.4+, Firefox 121+ (Baseline 2024+) */

/* Балансировщик нагрузки и Роутер */
body:has(> .http-request[data-endpoint="/auth"]) .auth-module {
  /* Перенаправляем "поток выполнения" в модуль авторизации */
  display: flex;
  transform: scale(1);
}
```

### Шаг 4: Бизнес-логика (ALU) с помощью CSS if()
**Это самое главное нововведение.** Начиная с Chrome 137, CSS поддерживает встроенную тернарную логику прямо в значениях свойств с помощью функции `if()`. В сочетании с **Range Syntax for Style Queries** (диапазонные запросы стилей), CSS может вычислять условия вида `>=`, `<=`, `and`, `or`.

```css
/* ПОДДЕРЖКА ВЕРСИЙ: 
   - if() function: Chromium 137+ (Август 2025)
   - Range Syntax for Style Queries: Chromium ~143+ (Конец 2025 - 2026) 
*/
.auth-module .css-processor-core {
  /* Вычисляем флаг авторизации на основе роли И возраста */
  --is-authorized: if(
    style(--req-role: admin): 1;
    style(--req-age >= 18): 1; /* Диапазонный запрос стилей (Range Syntax) */
    else: 0
  );

  /* Визуализация процесса для администратора сервера */
  background-color: if(
    style(--is-authorized: 1): #00ff00; /* Зеленый кубик - авторизован */
    else: #ff0000; /* Красный кубик - отказ */
  );

  /* Системный вызов (Syscall) - отправляем сигнал обратно в Tauri */
  animation: if(
    style(--is-authorized: 1): trigger-success-response 0.5s forwards;
    else: trigger-error-response 0.5s forwards;
  );
}
```

### Шаг 5: Сборка мусора (Garbage Collection) и ответ
Когда CSS заканчивает вычисление, отрабатывает анимация. Системная шина (JS) слушает стандартное событие DOM `animationend`. 
1. Если имя завершенной анимации `trigger-success-response`, JS считывает итоговые CSS-переменные узла (`getComputedStyle`).
2. JS формирует пакет `HTTP 200 OK` и отправляет по сети.
3. JS делает `.remove()` узлу DOM, освобождая оперативную память.

## 3. Таблица проверки версий и поддержки функционала (Mandatory Check)

Все описанные выше механики опираются на спецификации, принятые W3C и интегрированные в браузерные движки в период 2024–2026 годов:

| Технология / Фича | Статус в 2026 году | Минимальная версия движка | Назначение в CSS-Server |
| :--- | :--- | :--- | :--- |
| **CSS `if()` function** | Поддерживается | Chromium 137+, Edge 137+, Firefox (Nightly) | Ветвление логики (if/else) на уровне свойств без JS. |
| **CSS Style Queries** | Поддерживается | Chromium 119+, Safari 17.4+ | Позволяет функции `if()` ссылаться на значения `--variables` (состояния). |

| **Усовершенствованный `attr()`** | Поддерживается | Chromium 2026+ | Извлечение данных из атрибутов и приведение их к типам (`<number>`, `<color>`). |
| **`:has()` pseudo-class** | Baseline (Standard) | Chromium 105+, Safari 15.4+ | Паттерн-матчинг и роутинг «сверху вниз» по AST-дереву. |

## 4. Почему это не рухнет? (Математическая база)
Вам могут сказать, что CSS — это язык разметки. Но математически это уже не так.
Еще в 2010–2022 годах была доказана **Тьюринг-полнота** HTML+CSS (например, через реализацию клеточного автомата «Правило 110» или машины Тьюринга 2-state 2-symbol Busy Beaver на чистом CSS, где анимации играют роль тактового генератора процессора). С появлением в 2025 году `if()` и `style()` эта полнота перестала требовать безумных костылей из чекбоксов и селектора `~` — теперь это декларативная вычислительная среда.

## 5. Преимущества такого подхода (The "Glass Box")
1. **Абсолютная Визуализация:** Вы своими глазами видите, где скапливаются запросы (Bottle-necks). Если запросов много, Flexbox сожмет "блоки задач". Сервер физически "дышит" на экране.
2. **Нулевая десериализация в приложении:** Данные в памяти лежат прямо в DOM, их не нужно переводить в JSON для отладки. Чтобы проверить стейт бекенда — вы просто нажимаете <kbd>F12</kbd> (DevTools) в Tauri-контейнере и правите переменные руками прямо во время исполнения "кода".
